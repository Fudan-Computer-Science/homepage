- ### 本節重點及易錯細節
		- 在程式碼中「空白(space)」及「換行(enter)」對電腦而言是**沒有意義**的，因此一個程式設計者要**善用空白及換行整理程式碼**，**簡潔明瞭的程式碼**將是未來debug的關鍵。  
		- 而在程式碼中真正做為一項指令結束的是**分號(;)**，因此在每個命令下完記得要以**分號作結**。  
		- 忘記打分號: 程式會顯示 `[Error] expected ';' before 'XXX'` 簡單說，你少什麼，他就說他期望在XXX前有什麼  
		- `#include <iostream>`、`using namespace std;` 超級容易拼錯  

## 1-2 基本的輸出

## 小練習
- 對於以下程式碼，試問發生什麼錯誤?
(A) 語意錯誤
(B) 邏輯錯誤
(C) 語法錯誤
(D) 沒有錯誤
```cpp
#include<iostream>
with namespace std;
int main(){
    cout << "hello, world" << endl;
}
```
(題源: 109學年度入班考)

- 若執行時遇到以下錯誤訊息，試問可能犯了什麼錯誤?
`[Error] expected ';' before '}' token`
(A) 沒加`}`
(B) 沒加`;`
(C) 沒有宣告
(D) 沒有寫任何程式碼
(題源: 109學年度入班考)

- 試問以下程式碼是否可正常編譯並執行?
```
#include<iostream>
int main(){
    int a, b;
    std::cin >> a >> b        ;
    std::cout << a << " " << b << endl;
}
```
(A) 當然可以阿 還用想嗎
(B) 怎麼可能 沒有 `using namespace std;` 阿
(C) 不可能 第四行太多空白了
(D) 哈哈一定不行 因為" "內只能有一個字元
(題源: 109學年度入班考)
