時間複雜度


(出自<<我想成為影之強者！第二季>>15：00)
歡迎各位來到進階班~~~~
相信寫過二篩的人都被TLE搞得頭很痛，那我們要怎麼避免寫出會被題目TLE的code呢?
這時候我們就需要計算code的執行時間以比較code的優劣

計算​

假設只有運行 wah() 需要時間 111
下列兩個code哪個code較快呢?

for(int i=0;i<N;i++){
    for(int j=0;j<N;j++){
        wah();
    }
}
for(int i=0;i<N;i++){
    wah();
    wah();
    wah();
}

雖然在短期來看，上面的code會稍快，但對電腦來說，短期內的時間差距幾乎小到可以忽略不計，因此我們應該要注意的是這段code的趨勢。
故下面的code比較快。

big O 簡介​

O()O( )O()被用來描述一個code增長的上限，指的是這個code在最爛的情況下，執行時間的增長不會超過O()O()O()。

資訊

在計算O()O()O()時，要去忽略多項式的係數和常數，並取最高項次。
例如:
T(114514n2+810n+3)T(114514n^2+810n+3)T(114514n2+810n+3) = O(n2)O(n^2)O(n2)

以下是常見的時間複雜度和對應的演算法:

O(logn)O(logn)O(logn)對數階二分搜
O(n)O(n)O(n)線性階迴圈

O(n!)O(n!)O(n!)階層階排列組合，八皇后問題

執行時間​

那具體的執行時間呢，在電腦內直接看執行時間會因性能不同而有所差異，又大部分題目是以一秒為門檻，可參考以下表格。
