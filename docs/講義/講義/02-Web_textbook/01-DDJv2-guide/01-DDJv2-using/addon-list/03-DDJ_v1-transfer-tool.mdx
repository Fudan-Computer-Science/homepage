## 乾貨

### TypeScript 語法小教室
- `any vs unknown` 的差異
    1. `any` 是任意型別，也沒有型別檢查。型別檢查是 ts 有但 js 沒有的東西，可以防止一些設定問題。
    但如果要繞過玄學型別 bug 的話可以暫時使用。 
    (好像很多 AI 很喜歡 any，但一出錯就很難發現是型別問題，尤其用在資料讀取的地方。你永遠不知道使用者會塞什麼奇怪東西給你)
    
    2. `unknown` 叫作未知型別，有型別檢查。在使用之前要使用 `typeof` 語法來檢查型別，也可以做到強轉型態。
- 可選型別 `?`
    在這個 repo 的 index.ts 中，有 `static zjUrl?: string;` 這句。其中的「問號」就是所謂的可選型別。
    我們可以假設現在有一張問卷，要填寫國籍、性別和年齡，其中，性別和年齡都可以跳過不作答。
    在這個情況下，性別和年齡就是可選型別。
    `static zjUrl?: string;` 這樣寫是讓 `zjUrl` 這個變數變成可選。

- instanceof 做型別收窄才能安全存取屬性
