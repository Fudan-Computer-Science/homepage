## 1.2 競賽
### I. 賽制介紹
- XCPC (又名 ACM 賽制)
    1. 在賽中，所有題目一經提交馬上進行評分，以評分結果為準
    2. AC題目數多的排在前面，若AC題數相同，則以花費時間(含罰時)少的排在前
    3. 無法在比賽中查看錯誤的測資
    4. 賽中可不限次數提交
    5. 有多個測資點，要所有測資點全過才算AC
    6. 賽中可以看到大家的排名
    7. ICPC、Codeforce、Leetcode週賽皆為此種賽制
- OI
    1. 所有題目以最後一次繳交為準(編譯錯誤不計)
    2. 排名: 總分高的排名在前面，總分相等則排名相同
    3. 每一題提交後不會馬上評分(後測)
    4. 沒有罰時
- IOI
    1. 在賽中，所有題目一經提交馬上進行評分，以評分結果為準
    2. 多筆測資
    3. 沒有罰時
    4. 比賽可以看到排名，考試看不到排名
- 樂多
    1. 在賽中，參賽者可以多次提交一道題，但多次提交會導致得分被扣除。
    2. 排行榜按照參賽者的分數排
    3. 每題的最終得分為: \( s \cdot \max(0.95^n, 0.7) \)，其中，s為本次得分，n為本次提交前的嘗試次數
- IOI(嚴格)
    1. 比賽期間封榜，這意味著你在比賽期間不能看到其他人的分數。
    2. 分數計算為 在所有子任務中取最高得分並計算總和。

### II. 程式設計班常用模擬賽賽制設定
- APCS
    1. 特色: 後測、有子題分、沒有罰時
    2. 設置: OI賽制、每個級別三題且照難度排，Flexable Duration設定為120，Allow View Code取消打勾。
- TOI
    1. 直接沿用IOI賽制
- 學科能力競賽(北二)
    1. (交給草貓)
- 入班考/插班考實作
    1. 特色: 可以隨時查看排名、有子題分、沒有罰時
    2. 設置: IOI賽制，直接設定起始和結束時間即可。

## 1.3 功課
### 功課是甚麼?
功課特色如下:

1. 可以設定開始 & 結束日期
2. 可以設定遲交期限 (也就是說可以設定補題期限，也可以設定分數比例)
3. 如果你是程式設計班應屆幹部，請記得將作業管理員設定給其他教學/助教和負責計算學期總成績的人。

## 1.4 訓練

### I. 訓練是什麼?
簡單來說，訓練就是題單，這個功能的特色如下:

1. 未參加訓練計劃時不能檢視題目詳情。
2. 在題單內的題目不會被隱藏。這意味著任何人都可以在未參加訓練的情況下寫題。
3. 一個題單可以有很多章節
4. 各訓練之間是獨立的，沒有一定要完成哪個題單才能開始下一個。
