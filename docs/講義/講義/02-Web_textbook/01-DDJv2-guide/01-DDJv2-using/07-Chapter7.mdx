## 1.1 發現問題
若在使用DDJ-v2的過程中發現任何問題
- 若有能力可以嘗試自行修正並提出`Pull Request`
- 若有能力判斷
    - 若是HydroOJ本體的問題請至[HydroOJ Github](https://github.com/hydro-dev/Hydro)提出`Issue`
    - 若是DDJ-v2套件的問題請至[FDCS Github](https://github.com/Fudan-Computer-Science/ddj-v2-ui-setting)提出`Issue`
- 若無法判斷請聯繫當屆負責DDJ-v2的幹部協助判斷

## 1.2 成為DDJ-v2貢獻者
若想學習開發HydroOJ套件請至[DDJ-v2 套件撰寫手冊](/category/02-DDJv2-addon/)開始學習  
若想將自己開發的HydroOJ套件或已有的`npm packages`加入DDJ-v2成為官方套件，有兩種方案
1. 以`npm packages`的形式加入DDJ-v2
    0. 若用此方法建議要會維護該套件，因為後續若有問題需要更新或修正都需要有人負責
    1. 請先確認該`npm packages`是否可以在HydroOJ環境下運行
    2. 請至[FDCS Github](https://github.com/Fudan-Computer-Science/ddj-v2-ui-setting)中提出`Issue`，並附上
        - 該`npm packages`的名稱與連結
        - 簡單介紹(若有 UI 請附上截圖)
        - 使用說明文件連結(若有)
        - 為什麼你認為這個套件適合加入DDJ-v2
    3. 經過審核通過後，FDCS團隊會將該套件加入DDJ-v2 方法參照[上一章節](./06-Chapter6)  
2. 直接將自己開發的HydroOJ套件加入DDJ-v2
    0. 這個方法通常用於針對DDJ-v2進行特定功能擴充的套件
    1. 請先確認該套件的功能是否適合加入DDJ-v2
    2. 請至[FDCS Github](https://github.com/Fudan-Computer-Science/ddj-v2-ui-setting)中提出`Pull Request`，並附上
        - 簡單介紹(若有 UI 請附上截圖)
        - 使用說明文件連結(若有)
        - 為什麼你認為這個套件適合加入DDJ-v2
    3. 經過審核通過後，FDCS團隊會將該套件加入DDJ-v2 方法參照[上一章節](./06-Chapter6)

## 1.3 聯繫我們
若有任何問題或建議歡迎聯繫當屆負責DDJ-v2的幹部協助解決
也可以加入我們的[DDJ-v2 Discord 伺服器](https://discord.gg/fdcs)進行討論

## 1.4 功能請求
若有任何想要加入DDJ-v2的功能也歡迎提出功能請求
請至[FDCS Github](https://github.com/Fudan-Computer-Science/ddj-v2-ui-setting/issues)中提出`Issue`tag為`feature request`，並附上
- 簡單介紹該功能
- 為什麼你認為這個功能適合加入DDJ-v2
然後或許會在未來的版本中加入該功能
(也可以在下方留言區留言提出功能請求或DC伺服器中提出)
