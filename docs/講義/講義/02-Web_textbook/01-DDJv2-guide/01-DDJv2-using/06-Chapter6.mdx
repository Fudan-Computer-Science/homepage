# <span class="chapter_subtitle"> Management Methods </span>

## 1.1 什麼是好題目?
    首先，幹部域是以教學作為主要目的，競賽培訓為輔。因此，一題好的題目應當有明確的評量目標(教學/競賽/作業?)，
    並能扣合課綱。教學用題，同種類型的題目在出之前請看看學長姊有沒有出過類似的，不要讓整個題庫充盈了類似的題目。
    做為競賽題，應當標明其難度，以便未來計畫安排。隨著時代演進，競賽的範圍與難度都會略有變動，若為考古題修改應標明競賽名稱與年份。

## 1.2 什麼是好題解?
    一份好的題解應包含觀念提示及思路引導。非必要不得發具有可直接執行程式的題解。若在該題討論區發布與題目無關的討論，
    或發布具有危險性的可執行程式碼(例如引導使用者繞過電腦限制刪除System32)，應屆負責幹部有義務告知原發文者刪除。

## 1.3 行政管理辦法
    每屆程式設計班幹部應當由上一屆幹部遴選出一位負責人負責管理DDJ_v2。
    其人選應具備能長期且穩定的參與、能清楚回報技術問題、擅長自學的特質與能力。
    (註: 此職位需要的能力和競賽程式設計能力無絕對關係，會基本git/GitHub者佳) 

    負責內容如下:

    I. 管理該屆DDJ_v2系統，教不會使用的幹部使用其功能。

    II. 對於學員的需求能協助精確回報給開發組。

    III. 如系統發生臨時不能使用之情形，能判斷大致問題並依情況通知指導老師或開發組協助處理。


## 1.4 套件管理辦法
- 若有任何想要加入DDJ-v2的套件可以參照[第七章 成為貢獻者](./07-Chapter7)中的說明進行提案

### I. 審核流程
1. 提出申請
2. 幹部審核(功能適用性、維護性、穩定性等)
3. 幹部測試(功能測試、相容性測試等)
    - <span class="red_warning">試一定要實際操作套件並確認功能正常 幹部要負責測試套件 而不是單純看文件</span>
4. 通過審核後加入DDJ-v2

加入DDJ-v2請使用[hydrooj-addon-manager](./addon-list/hydrooj-addon-manager)進行安裝與管理套件
